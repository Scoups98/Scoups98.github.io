一、引言

生存分析是一种统计方法，用于分析从某个起点到特定事件发生的时间。在电信行业中，这种分析可以用来预测客户流失、硬件故障或服务升级等事件。本报告将详细描述如何使用生存分析处理电信客户数据，以预测客户流失时间，并分析每一步处理的原因。

二、数据处理流程及原因分析

环境配置与路径设置

配置数据处理的环境，包括导入必要的库和设置数据库连接。

import os

import getpass

from pyspark.sql.types import StructType, StructField, StringType, DoubleType

from pyspark.sql import SparkSession

from pyspark.sql.functions import when, col

import pandas as pd

import shutil

# 获取当前用户名（替代dbutils方法）

user = getpass.getuser() # 获取系统用户名

username_sql = user.replace(“.”, “_“) # 将用户名中的点替换为下划线，适配SQL语法

# 设置路径（使用本地路径而非DBFS路径）

database_name = f’dm_ibm_telco_churn_{username_sql}’ # 生成数据库名称

# 使用项目相对路径（建议存放在项目目录下）

project_root = os.path.abspath(os.path.join(os.getcwd(), “..”)) # 获取项目根目录（当前目录的上级目录）

data_path = os.path.join(project_root, ‘ibm-telco-churn’) # 定义数据存储的根路径

# 创建必要的目录结构

os.makedirs(data_path, exist_ok=True) # 创建数据根目录

os.makedirs(os.path.join(data_path, ‘bronze’), exist_ok=True) # 创建青铜层目录

os.makedirs(os.path.join(data_path, ‘silver’), exist_ok=True) # 创建白银层目录

# 定义完整文件路径

driver_to_dbfs_path = os.path.join(data_path, ‘Telco-Customer-Churn.csv’) # 定义数据文件路径

bronze_tbl_path = os.path.join(data_path, ‘bronze’) # 定义青铜层路径

silver_tbl_path = os.path.join(data_path, ‘silver’) # 定义白银层路径

# 表名保持不变

bronze_tbl_name = ‘bronze_customers’ # 青铜表名

silver_tbl_name = ‘silver_monthly_customers’ # 白银表名

# 打印验证配置

print(“配置验证：”)

print(f”用户名: {user}”)

print(f”SQL安全用户名: {username_sql}”)

print(f”数据库名: {database_name}”)

print(f”数据根目录: {data_path}”)

print(f”原始数据文件: {driver_to_dbfs_path}”)

print(f”Bronze层路径: {bronze_tbl_path}”)

print(f”Silver层路径: {silver_tbl_path}”)

print(f”Bronze表名: {bronze_tbl_name}”)

print(f”Silver表名: {silver_tbl_name}”)

```

为了使用PySpark进行数据处理，需要导入相关的库。通过getpass.getuser()获取系统用户名，确保数据库名称的唯一性和安全性。定义本地路径，确保数据处理的本地化和可移植性。之后创建必要的目录结构，确保数据的组织和存储清晰有序。

定义数据结构

定义数据结构，确保数据的正确加载和处理。

# Define schema

schema = StructType([

StructField(‘customerID’, StringType()), # 客户ID，字符串类型

StructField(‘gender’, StringType()), # 性别，字符串类型

StructField(‘seniorCitizen’, DoubleType()), # 是否为老年公民，双精度浮点类型（0或1）

StructField(‘partner’, StringType()), # 是否有配偶，字符串类型

StructField(‘dependents’, StringType()), # 是否有家属，字符串类型

StructField(‘tenure’, DoubleType()), # 在网时长，双精度浮点类型

StructField(‘phoneService’, StringType()), # 是否有电话服务，字符串类型

StructField(‘multipleLines’, StringType()), # 是否有多条线路，字符串类型

StructField(‘internetService’, StringType()), # 互联网服务类型，字符串类型

StructField(‘onlineSecurity’, StringType()), # 在线安全服务，字符串类型

StructField(‘onlineBackup’, StringType()), # 在线备份服务，字符串类型

StructField(‘deviceProtection’, StringType()), # 设备保护服务，字符串类型

StructField(‘techSupport’, StringType()), # 技术支持服务，字符串类型

StructField(‘streamingTV’, StringType()), # 是否有流媒体电视服务，字符串类型

StructField(‘streamingMovies’, StringType()), # 是否有流媒体电影服务，字符串类型

StructField(‘contract’, StringType()), # 合同类型，字符串类型

StructField(‘paperlessBilling’, StringType()), # 是否为无纸账单，字符串类型

StructField(‘paymentMethod’, StringType()), # 支付方式，字符串类型

StructField(‘monthlyCharges’, DoubleType()), # 每月费用，双精度浮点类型

StructField(‘totalCharges’, DoubleType()), # 总费用，双精度浮点类型

StructField(‘churnString’, StringType()) # 是否流失（字符串形式），字符串类型

])

通过定义数据结构，可以确保数据的正确加载和处理，避免数据类型不匹配的问题。

创建SparkSession

创建SparkSession，用于后续的数据处理。

# 创建SparkSession

spark = SparkSession.builder.appName(“TelcoCustomerChurn”).getOrCreate()

SparkSession是PySpark的入口点，用于创建和操作DataFrame，是数据处理的基础。

数据加载与显示：

从本地文件系统加载数据，并显示数据的前几行。

# 文件路径（假设文件已在本地Jupyter环境）

local_file_path = “Telco-Customer-Churn.csv” # 确保文件在Jupyter的工作目录中

# 读取数据

bronze_df = spark.read.format(‘csv’).schema(schema).option(‘header’, ‘true’).load(local_file_path)

# 显示数据

bronze_df.show(5)

从本地文件系统加载数据，确保数据的可用性。显示数据的前几行，验证数据是否正确加载。

数据清洗与转换

对原始数据集进行清洗和转换，以便进行生存分析。

# 构建白银表

silver_df = bronze_df.withColumn(‘churn’, when(col(‘churnString’) == ‘Yes’, 1)

.when(col(‘churnString’) == ‘No’, 0)

.otherwise(‘Unknown’)) \

.drop(‘churnString’) \

.filter(col(‘contract’) == ‘Month-to-month’) \

.filter(col(‘internetService’) != ‘No’)

# 查看白银表数据（可选）

silver_df.show(5)

将“churn”列转换为数值类型（0或1），简化后续的分析过程。过滤出订阅月对月合同且互联网服务不为“否”的客户，提高分析的针对性。

创建数据库

创建数据库，用于存储清洗后的数据。

# 设置数据库名称

database_name = “telco_db”

# 删除已存在的数据库（如果存在）

spark.sql(f’DROP DATABASE IF EXISTS {database_name} CASCADE’)

# 创建新数据库

spark.sql(f’CREATE DATABASE {database_name}’)

# 使用该数据库

spark.sql(f’USE {database_name}’)

print(f”数据库 {database_name} 创建并设置成功!”)

创建数据库对数据进行组织和管理，确保数据的结构化存储。如果存在旧的数据库，先删除它，以避免数据冲突。

数据写入本地文件系统

将清洗后的数据写入本地文件系统。

# 定义输出路径

bronze_output_path = “./data/bronze/Telco-Customer-Churn.csv”

silver_output_path = “./data/silver/Telco-Customer-Churn-processed.csv”

# 删除旧数据（如果存在）

shutil.rmtree(os.path.dirname(bronze_output_path), ignore_errors=True)

shutil.rmtree(os.path.dirname(silver_output_path), ignore_errors=True)

# 创建输出目录（如果不存在）

os.makedirs(os.path.dirname(bronze_output_path), exist_ok=True)

os.makedirs(os.path.dirname(silver_output_path), exist_ok=True)

# 使用 Spark 直接写入 CSV（避免转 Pandas，更高效）

bronze_df.write.csv(bronze_output_path, header=True, mode=”overwrite”)

silver_df.write.csv(silver_output_path, header=True, mode=”overwrite”)

print(f”数据已保存为CSV格式:\nBronze层: {bronze_output_path}\nSilver层: {silver_output_path}”)

将数据保存为CSV格式，便于后续的分析和报告生成。使用mode("overwrite")确保每次运行脚本时，数据都能被正确更新。

创建表

创建指向数据的表，以便于查询。

# 创建Bronze表（使用已定义的schema）

bronze_df.write.format(“csv”) \

.option(“header”, “true”) \

.mode(“overwrite”) \

.saveAsTable(f”{database_name}.{bronze_tbl_name}”)

# 创建Silver表（使用已定义的schema）

silver_df.write.format(“csv”) \

.option(“header”, “true”) \

.mode(“overwrite”) \

.saveAsTable(f”{database_name}.{silver_tbl_name}”)

# 验证表创建

print(“已创建表：”)

spark.sql(f”SHOW TABLES IN {database_name}”).show()

# 验证数据可查询

print(“\nBronze表数据示例：”)

spark.sql(f”SELECT * FROM {database_name}.{bronze_tbl_name} LIMIT 5”).show()

print(“\nSilver表数据示例：”)

spark.sql(f”SELECT * FROM {database_name}.{silver_tbl_name} LIMIT 5”).show()

通过创建表将数据结构化，便于在SQL查询中使用。接下来验证表是否成功创建，确保数据的可访问性。

四、总结

通过上述步骤，我们成功地配置了数据处理环境，下载并加载了数据，进行了数据清洗和转换，创建了数据库和表，并将数据写入了本地文件系统。这些步骤为后续的生存分析奠定了基础，使我们能够预测客户流失时间，从而帮助电信公司制定更有效的客户保留策略。

关键：

  1. 数据清洗和转换是确保分析准确性的关键步骤。
  2. Kaplan-Meier估计和Cox比例风险模型是生存分析的核心方法。
  3. 模型评估和验证确保了分析结果的可靠性和可重复性。
  4. 生存分析的结果可以为实际问题的决策提供有力支持。

通过这些步骤，可以确保数据处理的透明度和可重复性，为生存分析提供高质量的数据支持。
